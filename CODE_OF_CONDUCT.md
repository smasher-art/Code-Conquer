# Code of Conduct

> **Project:** Bebo Tech — Soulslike Learning Platform
> **BTES Ambassador Program 2025 · Chitkara University**
> Adapted from the [Contributor Covenant](https://www.contributor-covenant.org/), version 2.1.

---

## Our Pledge

We, as members, contributors, and leaders of the Bebo Tech project — as part of the BTES Ambassador Programme at Chitkara University — pledge to make participation in our project and community a **harassment-free experience for everyone**, regardless of age, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

We pledge to act and interact in ways that contribute to an **open, welcoming, diverse, inclusive, and healthy community** — both in our technical collaboration and in the spirit of the BTES programme.

---

## Our Standards

### Positive behaviours we encourage

- **Be kind and constructive.** Give feedback on code and ideas, not on people.
- **Help others learn.** This is a student programme. Not everyone has the same experience level — that is the point.
- **Acknowledge mistakes gracefully.** Apologise when appropriate and learn from the experience.
- **Use inclusive language.** Write commit messages, comments, PR descriptions, and issues that anyone can understand and feel welcome reading.
- **Respect the learning process.** The Soulslike philosophy — *attempt, fail, refine, overcome* — applies to coding and to collaboration.
- **Give credit generously.** Acknowledge ideas, code snippets, and resources that influenced your work.
- **Accept constructive criticism.** A PR review that requests changes is a sign someone cares about the project quality, not a personal attack.

### Unacceptable behaviours

- Harassment, intimidation, or discrimination of any kind — in code reviews, issues, PRs, or any project-related communication.
- Sexualised language, imagery, or advances.
- Trolling, personal attacks, or derogatory comments.
- Publishing someone's private information (email, phone, address) without their explicit consent.
- Dismissing or belittling questions or contributions because someone is a beginner.
- Plagiarism — presenting others' code or work as your own without attribution.
- Deliberate introduction of security vulnerabilities or malicious code.
- Sustained disruption of project discussions, code reviews, or programme activities.
- Violating any Chitkara University academic integrity policies in the context of this programme.

## Academic Integrity (Programme-Specific)

As this project is submitted as part of the **BTES Ambassador Programme**:

- All code, documentation, and assets submitted must represent the original work of the team, or must clearly attribute any third-party sources (libraries, tutorials, sample code).
- Using AI code-generation tools is permitted for assistance and exploration, but final submissions must represent the team's understanding. Do not submit AI-generated code without reviewing and understanding it.
- Any external dataset, image, or resource must be used in compliance with its licence.
- Violations of academic integrity will be reported to the programme coordinators.

## Enforcement Responsibilities

Project maintainers (team leads and BTES mentors) are responsible for clarifying and enforcing the standards of acceptable behaviour. They may take corrective action in response to any behaviour they deem inappropriate, threatening, offensive, or harmful.

Maintainers have the right and responsibility to remove, edit, or reject comments, commits, code, documentation edits, issues, and other contributions that are not aligned with this Code of Conduct, and will communicate reasons for moderation decisions when appropriate.

---

## Scope

This Code of Conduct applies within all project spaces — GitHub issues, pull requests, code comments, the project wiki, and any other project-related communication channels. It also applies when an individual is officially representing the project in public, for example at a college event or in a social media post referencing the programme.

## Enforcement Guidelines

Project maintainers will follow these Community Impact Guidelines to determine the appropriate response:

### 1 — Correction

**Impact:** A minor first offence; use of inappropriate language or behaviour deemed unprofessional.
**Response:** A private written warning explaining the nature of the violation and expected correction.

### 2 — Warning

**Impact:** A single significant incident or a pattern of minor incidents.
**Response:** A formal warning with consequences for continued behaviour. A temporary restriction from project interaction may be applied.

### 3 — Temporary suspension

**Impact:** A serious violation of community standards or sustained inappropriate behaviour.
**Response:** Temporary suspension from contributing to the project (PRs, issues, discussions) for a defined period. A return is conditional on agreed improvements.

### 4 — Permanent removal

**Impact:** A pattern of violations, severe single incident (e.g. harassment, plagiarism), or behaviour that fundamentally undermines the programme's integrity.
**Response:** Permanent removal from project participation. This will be escalated to the BTES programme coordinators and, where relevant, to Chitkara University's academic integrity body.
